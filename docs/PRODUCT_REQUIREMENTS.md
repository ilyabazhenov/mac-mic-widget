# Требования к продукту mac-mic-widget (реализовано)

Документ фиксирует требования к продукту, которые на данный момент реализованы в кодовой базе.

## 1. Цель продукта

Дать быстрый способ выключать/включать микрофон в macOS пользователям, у которых нет аппаратной кнопки mute/unmute на гарнитуре.

## 2. Платформа и формат

- Тип приложения: утилита в строке меню macOS
- Целевая платформа: macOS 13+
- Модель запуска: фоновое приложение со status bar item и popover UI

## 3. Ключевые функциональные требования

### FR-1: Показывать текущий уровень микрофона в строке меню

- В status bar item отображаются:
  - иконка микрофона (`mic.fill` или `mic.slash.fill`)
  - текущий input level в процентах (`0...100%`)
- Значение обновляется почти в реальном времени через периодический refresh (`0.5s`).

### FR-2: Быстро переключать микрофон on/off

- Поведение переключения:
  - Если текущий input volume > 0, установить `0`.
  - Если текущий input volume == 0, восстановить предыдущий ненулевой уровень.
- Fallback восстановления:
  - Если предыдущий ненулевой уровень неизвестен/слишком мал, восстанавливать минимум до `5%`.

### FR-3: Разделить левый и правый клик

- Левый клик по status item:
  - открывает/закрывает popover.
- Правый клик (и `ctrl + left click`):
  - напрямую выполняет mute/unmute,
  - не должен открывать popover.

### FR-4: Информация и элементы управления в popover

Popover содержит:

- текущий процент уровня (крупное числовое значение),
- цветной бейдж уровня (`Muted` / `Low` / `Medium` / `High`),
- progress bar на всю ширину контента,
- основную кнопку действия на всю ширину (`Mute microphone` / `Unmute microphone`) со стабильной шириной,
- кнопку `Quit`.

## 4. Визуальные требования

### VR-1: Цвет иконки по уровню

Цвет иконки в status bar зависит от уровня:

- muted -> secondary label color,
- `< 30%` -> red,
- `30...60%` -> yellow,
- `> 60%` -> green.

### VR-2: Стабильность layout в status bar и popover

- Ширина текста в status bar достаточно зафиксирована, чтобы избежать заметных скачков при изменении процента.
- Ширина кнопки mute/unmute в popover фиксирована (full-width), чтобы не было скачков между состояниями.

## 5. Поведение аудио I/O

### AR-1: Читать уровень микрофона сначала из системных настроек

- Основной источник input volume:
  - AppleScript `input volume of (get volume settings)` через `osascript`.
- Значение интерпретируется как `0...100` и конвертируется в `0...1`.

### AR-2: Fallback на CoreAudio

Если script-путь недоступен/ошибается, используется CoreAudio fallback:

- определение default input device,
- чтение scalar volume по набору candidate elements/channels,
- попытка записи во все доступные settable elements.

### AR-3: Polling и синхронизация состояния

- Сервис опрашивает уровень каждые `0.5s` через timer в `RunLoop.main` с режимом `.common`.
- После любого явного toggle выполняется немедленный refresh состояния.

## 6. Требования к обработке ошибок

- В popover показывается текст ошибки, если read/write завершился неуспешно.
- Приложение не должно завершаться при сбое отдельной операции чтения/записи.

## 7. Требования к developer experience

- `make run`: запуск приложения
- `make dev`: watch + restart при изменениях файлов (требует `watchexec`)
- `make test`: запуск тестов
- `make build`, `make clean`, `make deps`: служебные команды

## 8. Известные ограничения

- Поддержка writable input gain через CoreAudio зависит от устройства.
- Работа AppleScript-пути зависит от поведения системных аудио-настроек.
- Некоторые гарнитуры/аудиоинтерфейсы используют нестандартные контролы gain.
